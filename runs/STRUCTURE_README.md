# Example structure of `runs/segment/` after training

## Assumptions (current in this workspace)
- Outputs were generated by `scripts/local/train_models.py`.
- Under `runs/segment/`, each `{model_name}` has a timestamp folder.
- Inside each timestamp folder, each experiment (`no_augmentation`, `baseline`, `moderate_geom`, `strong_geom`, optional `strong_geom_fruit`) has an output folder plus `VAL_*` folders for qualitative validation visuals.

## Structure (current in this workspace):
```
runs/
└── segment/
    └── {model_name}/
        └── {timestamp}/
            ├── {experiment}/
            │   ├── weights/
            │   │   ├── best.pt
            │   │   └── last.pt
            │   ├── results.csv
            │   ├── results.png
            │   ├── confusion_matrix*.png
            │   ├── *F1_curve*.png / *PR_curve*.png / ...
            │   └── train_batch*.jpg
            └── VAL_{experiment}/
                ├── confusion_matrix*.png
                ├── val_batch*_labels.jpg
                └── val_batch*_pred.jpg
```

## Quick summary (for new users)
- `{timestamp}/` groups all experiments from a single training run.
- `{experiment}/` contains training outputs, including `weights/` and `results.csv` (this is where the numeric metrics live).
- `VAL_{experiment}/` mainly contains visuals (e.g. `val_batch*_labels.jpg` and `val_batch*_pred.jpg`). In your workspace it typically does not contain `results.csv`.

## Where are the metrics used for comparison?
`compare_models.py` / `compare_models_demo_day.py` read from:
- `runs/segment/{model_name}/{timestamp}/{experiment}/results.csv`

## Example paths
- best weights: `runs/segment/{model_name}/{timestamp}/{experiment}/weights/best.pt`
- validation visuals: `runs/segment/{model_name}/{timestamp}/VAL_{experiment}/val_batch*_pred.jpg`
